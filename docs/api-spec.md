# MindTrace AI – Backend API Specification

This document describes the REST APIs exposed by the **MindTrace AI** backend.  
All endpoints are designed with **security, clarity, scalability, and ethical AI principles** in mind.

> ⚠️ Note: MindTrace AI is an **early cognitive stress & burnout indicator**, not a medical or diagnostic system.

---

## Authentication

Authentication is handled using **token-based security**.

- Passwords are hashed
- Tokens are required for protected endpoints
- No sensitive personal data is exposed

---

## API Endpoints Overview

| Category        | Endpoint                     | Method | Description |
|-----------------|------------------------------|--------|-------------|
| Authentication  | `/auth/register`             | POST   | Register a new user |
| Authentication  | `/auth/login`                | POST   | Authenticate user |
| Assessment      | `/assessment/submit`         | POST   | Submit cognitive & behavioral data |
| Insights        | `/insights/risk-score`       | GET    | Get stress & burnout risk score |
| Insights        | `/insights/recommendations`  | GET    | Get AI-driven recommendations |
| User            | `/user/profile`              | GET    | Fetch user profile |

---

## 1. Register User

### Endpoint
- POST /auth/register


### Description
Creates a new user account with secure credential storage.

### Request Body
```json
{
  "name": "User Text",
  "email": "user@example.com",
  "password": "strong_password"
}
```
### Response
```
{
  "message": "User registered successfully"
}
```

## 2. Login User

### Endpoint
- POST /auth/login

### Description
Authenticates user and returns an access token.

### Request Body
```
{
  "email": "user@example.com",
  "password": "strong_password"
}
```
### Response
```
{
  "access_token": "jwt_token_here",
  "token_type": "bearer"
}
```

## 3. Submit Assessment Data

### Endpoint
- POST /assessment/submit

### Description
Accepts cognitive and behavioral inputs for AI-based stress analysis.

### Headers
Authorization: Bearer <token>

### Request Body
```
{
  "focus_level": 6,
  "sleep_hours": 5,
  "task_completion_time": 42,
  "self_reported_mood": 3
}
```
### Response
```
{
  "message": "Assessment data submitted successfully"
}
```

## 4. Get Risk Score

### Endpoint
- GET /insights/risk-score

### Description
Returns the calculated cognitive stress & burnout risk score generated by the ML model.

### Headers
Authorization: Bearer <token>

### Response
```
{
  "stress_score": 0.78,
  "risk_level": "High"
}
```

## 5. Get Recommendations

### Endpoint
- GET /insights/recommendations

### escription
Provides personalized, non-clinical, preventive recommendations.

### Headers
Authorization: Bearer <token>

### Response
```
{
  "recommendations": [
    "Take short breaks during long tasks",
    "Maintain consistent sleep routines",
    "Reduce multitasking where possible"
  ]
}
```

## 6. Get User Profile

### Endpoint
- GET /user/profile

### Description
Fetches basic user profile details.

### Headers
Authorization: Bearer <token>

### Response
```
{
  "name": "User Text",
  "email": "user@example.com",
  "joined_on": "2026-01-10"
}
```

## Error Handling
All errors follow a consistent format.

### Example Error Response
```
{
  "error": "Unauthorized",
  "message": "Invalid or expired token"
}
```

## Security & Privacy Considerations

- No raw behavioral data is exposed externally
- All communications occur over secure channels
- Data is anonymized for ML processing
- The system does not provide medical diagnoses

### Tech Stack Reference

- **Backend:** FastAPI (Python)
- **Authentication:** Token-based (JWT)
- **ML Model:** Random Forest
- **Database:** SQLite
- **Deployment:** Docker-ready

## Disclaimer

MindTrace AI is an early-warning and awareness tool designed to promote well-being.
It should not be used as a substitute for professional medical or psychological advice.
